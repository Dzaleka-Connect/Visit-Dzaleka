Contributing Guide

Thank you for your interest in contributing to Visit Dzaleka! As an open-source project, we welcome contributions from the community to help improve the visitor management system for Dzaleka Refugee Camp.

Getting Started

  1. Fork the repository on GitHub
  2. Clone your fork locally
  3. Create a new branch for your feature or fix: git checkout -b feature/amazing-feature
  4. Install dependencies: npm install
  5. Start development server: npm run dev

Development Guidelines

Code Style

  • We use TypeScript for type safety. Avoid using any whenever possible.
  • Follow the existing project structure and naming conventions.
  • Use Tailwind CSS for styling. Avoid writing custom CSS unless absolutely necessary.
  • Ensure all components are responsive and accessible.

Database Changes

If your change involves the database schema:

  1. Modify shared/schema.ts to define the new tables or columns.
  2. Generate a migration or update the schema using Drizzle Kit.
  3. Include any necessary SQL migration files in your PR.

Adding New Features

When adding a new feature:

  1. Plan: Create a brief design doc or issue describing the feature.
  2. Backend: Implement the API endpoints in server/routes.ts and database logic in server/storage.ts.
  3. Frontend: Create the UI components in client/src/components and pages in client/src/pages.
  4. Test: Verify the feature works as expected and handles edge cases.

Pull Request Process

  1. Push your changes to your fork.
  2. Submit a Pull Request to the main repository.
  3. Provide a clear description of the changes and the problem they solve.
  4. Include screenshots or videos for UI changes.
  5. Wait for review and address any feedback.

Community & Support

If you have questions or need help, please open an issue on GitHub or join our community discussions.

Tip: Check the "Good First Issue" label on GitHub for beginner-friendly tasks to get started with!
